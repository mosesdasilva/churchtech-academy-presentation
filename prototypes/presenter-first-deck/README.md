# Presenter-First Prototype

This prototype explores how the ChurchTech Academy proposal would feel if the live deck were intentionally sparse and the detailed explanation lived in a separate document.

## How to run

Open `prototypes/presenter-first-deck/index.html` directly in a browser.

## Prototype intent

- keep the current dark blue presentation language
- reduce on-slide wording so the presenter carries more of the explanation verbally
- preserve the same ministry proposal structure at a higher level
- move the fuller supporting content into `presentation-notes.md`

## Notes

- Root project files remain unchanged.
- This is a prototype branch and folder, not a replacement for the main deck.
- The companion notes document is meant to stand in for the fuller written submission or presenter notes.
